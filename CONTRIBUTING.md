# Contributing

Thanks for considering a contribution!

## Quick Start

1. Fork the repo and create a feature branch.
2. Install dependencies in the root and `frontend/`.
3. Run the API and frontend locally.

## Development Notes

- Keep changes focused and small.
- Prefer clear, readable code over cleverness.
- Add or update docs when behavior changes.

## Commit Messages

Use short, descriptive commit messages (e.g., "Fix auth token refresh").

## Pull Requests

- Describe the problem and the solution.
- Include steps to verify the change.
- Call out any follow-up work or known issues.
